# B2B 平台项目开发流程（多 Agent 协作）

基于 claude-code-guide 的思维方式，结合 OpenClaw 的多智能体编排能力，适配 5-6 人团队与 B2B 平台交付。

## 1. 输入物
- 原型：Axure
- 数据库：PowerDesigner PDM 或直连 MySQL 设计

## 2. 角色与职责（人+Agent）
- 产品负责人：需求澄清/范围控制
- 架构负责人：技术选型/架构方案
- 全栈工程师：业务开发
- 后端专家：API/数据建模
- QA：测试与质量
- DevOps：部署与上线

### Agent 集群（对应工作流）
- Planner：需求拆解、里程碑
- Architect：架构设计、接口与数据模型
- Code Reviewer：代码规范/风险审查
- Security Reviewer：安全/权限/依赖审查
- Test Runner：测试策略/覆盖率
- Doc Updater：文档与交付物更新

## 3. 全流程（从需求到上线）
1) 需求接入
   - 输入：PRD、原型、PDM/DB
   - 输出：需求澄清清单 & 风险点

2) 需求评审与拆解
   - Planner 产出：功能清单、优先级、里程碑
   - 输出：需求拆解表（含变更记录）

3) 架构设计
   - Architect 产出：模块边界、API 设计、数据模型
   - 输出：架构说明 + 接口草案

4) 任务执行
   - 工程师按模块开发
   - Code Reviewer 进行 PR 审查

5) 测试与验收
   - Test Runner 产出测试计划与覆盖率建议
   - QA 输出测试报告

6) 部署上线
   - DevOps 生成部署记录与回滚预案
   - 上线后监控与回归检查

7) 操作说明与知识库
   - Doc Updater 汇总交付物
   - 知识库归档

## 4. 交付物清单
- PRD / 需求说明
- 需求处理过程记录
- 架构与接口设计
- 测试报告
- 部署上线说明
- 操作说明
- 知识库
