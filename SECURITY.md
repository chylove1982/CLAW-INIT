# 安全规范（参考 claude-code-guide）

## 1. 基本原则
- **最小权限**：只开放必要工具/权限
- **外部操作需确认**：发布/权限/消息推送必须人工确认
- **敏感信息不入库**：密钥/证书/私钥禁止进入仓库

## 2. Hook/自动化安全（适配 OpenClaw）
- 始终对输入进行校验与转义
- 禁止对 `~/.ssh`、`.env`、`.git` 等敏感路径写入
- 任何自动执行脚本必须带白名单

## 3. 常见安全检查项
- 注入风险（SQL/XSS/命令注入）
- 认证/授权边界
- 日志脱敏
- 依赖漏洞（CVE）

## 4. 推荐流程
- 需求阶段：列出安全风险
- 开发阶段：Security Reviewer 审查
- 上线前：安全清单逐项确认
