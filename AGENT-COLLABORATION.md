# 多 Agent 协作细则

## 1. 输入/输出规范
- Planner：输入 PRD/原型/PDM → 输出任务拆解+里程碑
- Architect：输入需求+约束 → 输出架构/接口/数据模型
- Developer：输入任务清单 → 输出代码/PR
- Reviewer：输入 PR → 输出问题清单
- Security：输入代码/配置 → 输出安全审查结论
- QA/Test：输入功能列表 → 输出测试报告
- Doc Updater：输入变更记录 → 输出交付文档

## 2. 协作节奏
- 每个阶段产出可复核的文档
- 未通过审查不得进入下一阶段

## 3. 验收标准
- 功能通过测试
- 文档齐全
- 部署可回滚
